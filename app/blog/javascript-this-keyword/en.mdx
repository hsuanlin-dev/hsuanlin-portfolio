# A Complete Guide to JavaScript `this`: 5 Binding Rules and Real-World Usage

In JavaScript development, the `this` keyword is one of the core mechanisms behind object-oriented behavior and function invocation. Its value is determined dynamically based on how a function is called, which often leads to bugs when dealing with object methods, event listeners, asynchronous callbacks, or constructors.

This article provides a comprehensive breakdown of the five binding rules of `this`, their precedence, the differences between `call`, `apply`, and `bind`, and the key distinctions between arrow functions and regular functions. By understanding these concepts, you can control `this` more precisely and write more reliable, maintainable code.

---

## The Five Binding Rules of `this`

In JavaScript, `this` is a special keyword that refers to the "execution context object" of a function. The core principle is simple:

> **Who calls the function determines what `this` refers to.**

The five binding rules are:

- **Default Binding**: When a function is called directly, `this` refers to the global object (in non-strict mode) or `undefined` (in strict mode).
- **Implicit Binding**: When a function is called as a method of an object, `this` refers to that object.
- **Explicit Binding**: Using `call()`, `apply()`, or `bind()` to explicitly set `this`.
- **New Binding**: When a function is called with `new`, `this` refers to the newly created instance.
- **Arrow Function Binding**: Arrow functions do not have their own `this`; they inherit it from the surrounding scope.

---

To make these concepts easier to understand, let's use a simple analogy:

Think of `this` as a **spotlight**—it shines on different people depending on the situation.

---

### 1. Default Binding — **"Random Stranger Mode"**

When a function is called independently (without an object, `new`, or explicit binding), default binding applies.

- **Scenario**: You shout "Hey!" in public without addressing anyone.
- **Explanation**: The function is called directly, so the spotlight has no specific target. It shines on the "whole world" (the global object). In strict mode, the light simply turns off (`undefined`).
- **Analogy**: You shout "handsome guy!" in a plaza—no one responds, just empty air.
- In non-strict mode, `this` refers to the global object (`window` in browsers, `global` in Node.js).
- In strict mode, `this` is `undefined`.

```js
function showThis() {
  console.log(this);
}
showThis(); // window (non-strict mode)

'use strict';
function strictShow() {
  console.log(this);
}
strictShow(); // undefined
```

### 2. Implicit Binding — **"Belonging to a Group Mode"**

When a function is called as a method of an object (`obj.method()`), `this` refers to the object before the dot.

- **Scenario**: You're an artist under a talent agency, and the company introduces you as "our artist."
- **Explanation**: When a function is invoked as a method, `this` automatically points to the object that owns the method.
- **Analogy**: If someone refers to "the boss of this company," the spotlight shines on the company's owner.

```js
const user = {
  name: 'Bruce',
  greet() {
    console.log(`Hello, I am ${this.name}`);
  }
};

user.greet(); // Hello, I am Bruce
```

Note: If the method is extracted and called independently, implicit binding is lost and falls back to default binding.

```js
const greetFn = user.greet;
greetFn(); // undefined (or window)
```

### 3. Explicit Binding — **"Directly Assigned Mode"**

Using `call()`, `apply()`, or `bind()` allows you to explicitly define what `this` should be.

- **Scenario**: You belong to Company A, but Company B specifically calls you in for a role.
- **Explanation**: You manually control where the spotlight points.
- **Analogy**: A director points and says, "You're the lead," regardless of your original affiliation.

### 4. `new` Binding — **"New Identity Mode"**

When a constructor function is called with the `new` keyword, JavaScript creates a new empty object, binds `this` to that object, and automatically returns it.

- **Scenario**: A company is holding an audition to "create" a brand-new idol.
- **Explanation**: When a constructor function is invoked with `new`, a new empty object is created, and `this` is bound to this newly created object.
- **Analogy**: A new artist just debuted—the spotlight shines on this newly born star.

```js
function Person(name) {
  this.name = name;
}

const bruce = new Person('Bruce');
console.log(bruce.name); // Bruce
```

### 5. Arrow Function Binding — **"Inherited Spotlight Mode"**

Arrow functions don't have their own `this`. Instead, they inherit `this` from their surrounding lexical scope. This binding cannot be changed. This makes them especially useful in callbacks and nested functions to avoid losing the correct `this`.

- **Scenario**: You don't have your own spotlight—you stand wherever your manager stands.
- **Explanation**: `this` is determined at definition time, not invocation time. Once set, even `call` or `bind` cannot change it.
- **Analogy**: Wherever your team leader goes, you follow.

```js
const obj = {
  name: 'Bruce',
  delayedGreeting() {
    setTimeout(() => {
      console.log(`Hello, ${this.name}`); // Hello, Bruce
    }, 1000);
  }
};
obj.delayedGreeting();
```

## `this` Binding Priority (from Highest to Lowest)

When determining `this`, follow this order of precedence—the higher the rule, the earlier it applies:

1. **Arrow Functions**: Use the `this` from the surrounding lexical scope at definition time, unaffected by other rules.
2. **New Binding**: When a constructor function is called with `new`.
3. **Explicit Binding**: When using `call()`, `apply()`, or `bind()`.
4. **Implicit Binding**: When a function is called as a method of an object.
5. **Default Binding**: The fallback rule when none of the above apply.

## Differences and Use Cases of `call`, `apply`, and `bind`

| Method | Execution | Argument Passing | Common Use Cases |
| --- | --- | --- | --- |
| `call` | Executes immediately | Passed individually | Borrowing methods from other objects, calling parent constructors |
| `apply` | Executes immediately | Passed as an array | Handling multiple arguments (e.g., `Math.max`), dynamic arguments |
| `bind` | Returns a new function | Can preset partial arguments | Event listeners, fixing `this` in callbacks |

```js
function introduce(hobby1, hobby2) {
  console.log(`I am ${this.name}, I like ${hobby1} and ${hobby2}`);
}

const person = { name: 'Bruce' };

introduce.call(person, 'coding', 'reading');    // executes immediately
introduce.apply(person, ['coding', 'reading']); // executes immediately
const bound = introduce.bind(person, 'coding');
bound('reading'); // returns a new function, then invoked later

// Output:
// I am Bruce, I like coding and reading
// I am Bruce, I like coding and reading
// I am Bruce, I like coding and reading
```

---

## **Differences Between Arrow Functions and Regular Functions (`this`)**

- **Arrow Functions**: Do not have their own `this`. They inherit `this` from the surrounding lexical scope and cannot be changed using `call`, `apply`, or `bind`.
- **Regular Functions**: `this` is determined by how the function is called and can be changed using the three methods above.

```js
const obj = {
  name: 'object',
  regularMethod() {
    setTimeout(function() {
      console.log(this.name); // undefined (`this` becomes window)
    }, 100);
  },
  arrowMethod() {
    setTimeout(() => {
      console.log(this.name); // 'object' (inherits outer `this`)
    }, 100);
  }
};

obj.regularMethod();
obj.arrowMethod();

// Output:
// undefined
// object
```

## Determine the Value of `this` in the Following Examples:

```js
// Example 1: Implicit binding
const obj = { name: 'object', method() { console.log(this.name); } };
obj.method(); // 'object'

// Example 2: Method extraction
const method = obj.method;
method(); // undefined (default binding)

// Example 3: Arrow function callback
const arrowObj = {
  name: 'arrow',
  method() {
    setTimeout(() => {
      console.log(this.name);
    }, 100);
  }
};
arrowObj.method(); // 'arrow'

// Example 4: new binding
function Person(name) {
  this.name = name;
  this.sayHi = function() {
    console.log('Hi, ' + this.name);
  };
}
const person = new Person('Bruce');
person.sayHi(); // 'Hi, Bruce'
```

## **Best Practices**

- For callbacks (e.g., `setTimeout`, `addEventListener`), prefer arrow functions to avoid incorrect `this` binding.
- In classes, if a method is used as a callback, bind it in the constructor (`this.method = this.method.bind(this)`), or define it as an arrow function property.
- When debugging `this`, always ask: **"Who is calling this function, and how is it being called?"** Then apply the five binding rules and their priority to quickly identify the issue.
