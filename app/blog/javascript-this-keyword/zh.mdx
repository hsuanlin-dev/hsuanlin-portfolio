# JavaScript this 關鍵字徹底解析

在 JavaScript 開發中，this 關鍵字的指向是物件導向與函式呼叫的核心機制之一。它會根據不同的呼叫情境動態決定指向哪個物件，許多開發者在處理物件方法、事件監聽器、異步回調或建構函式時，都曾因為 this 指向錯誤而遇到 Bug。

以下完整說明 this 的五大綁定規則、判斷優先順序、call/apply/bind 的差異，以及箭頭函式與一般函式的關鍵不同。透過這些內容，可以更精準地掌控 this 的行為，讓程式碼更可靠且易於維護。

---

## this 的五大綁定規則

JavaScript 中的 this 是一個特殊關鍵字，指向函式執行時的「上下文物件」。理解 this 的核心原則是：誰調用函式，this 就指向誰。

- `預設綁定`：直接調用函式時，this 指向全域物件（非嚴格模式）或 `undefined`（嚴格模式）
- `隱含綁定`：作為物件方法調用時，this 指向調用該方法的物件
- `顯式綁定`：使用 `call()`、`apply()`、`bind()` 明確指定 this 的值
- `new 綁定`：使用 `new` 關鍵字調用函式時，this 指向新建立的物件實例
- `箭頭函式綁定`：箭頭函式沒有自己的 this，使用外層作用域的 this

---

以下用**「追星／追偶像」**這個簡單的比喻，把 `this` 的五大用法梳理清楚：

想像 `this` 就是一個**「聚光燈」**，它會根據情況照在不同的人身上：

---

### 1. 預設綁定 (Default Binding) —— **路人模式**

當函式以獨立方式直接呼叫（沒有物件前綴），且沒有使用 new、call、apply 或 bind 時，就會觸發預設綁定。

- **情境**：你在路邊隨便大喊一聲「喂！」，沒有針對任何人。
- **解釋**：直接呼叫一個普通的函式。這時聚光燈沒目標，只好照向整個大地（全域物件 `window`），如果是嚴格模式，燈就直接關了（`undefined`）。
- **比喻**：在廣場大喊「帥哥」，沒人理你，只有空氣。
- 非嚴格模式下，this 指向全域物件（瀏覽器為 window，Node.js 為 global）。
- 嚴格模式下，this 為 undefined。

```js
function showThis() {
  console.log(this);
}
showThis(); // window（非嚴格模式）

'use strict';
function strictShow() {
  console.log(this);
}
strictShow(); // undefined
```

### 2. 隱含綁定 (Implicit Binding) —— **所屬團體模式**

當函式作為物件的方法被呼叫時（`obj.method()`），`this` 會自動指向「點」前面的物件。

- **情境**：你是某個經紀公司（物件）旗下的藝人，當公司介紹「**我們的**藝人」時。
- **解釋**：函式作為物件的一個方法被呼叫（例如 `obj.method()`），這時 `this` 就指向那個**點（.）**前面的物件。
- **比喻**：周杰倫在「杰威爾音樂」裡，所以提到「這間公司的老闆」，聚光燈就會打在周杰倫身上。

```js
const user = {
  name: 'Bruce',
  greet() {
    console.log(`Hello, I am ${this.name}`);
  }
};

user.greet(); // Hello, I am Bruce
```

注意：如果將方法抽離出來單獨執行，隱含綁定就會失效，退回預設綁定：

```js
const greetFn = user.greet;
greetFn(); // undefined（或 window）
```

### 3. 顯式綁定 (Explicit Binding) —— **指定點名模式**

透過 `call()`、`apply()`、`bind()` 明確指定 `this` 的指向，適合需要借用其他物件方法或固定執行環境的場景。

- **情境**：雖然你是 A 公司的藝人，但今天 B 公司「點名」要你來支援。
- **解釋**：使用 `call`、`apply` 或 `bind`。你強行指定聚光燈要照在哪個物件身上。
- **比喻**：導演直接點名：「我要那個人（物件）當主角」，不管他本來是誰。

### 4. `new` 綁定 (new Binding) —— **打造新人模式**

使用 `new` 關鍵字呼叫建構函式時，JavaScript 會先建立一個新空物件，並將 `this` 綁定到這個新物件上，同時自動返回該物件。

- **情境**：公司正在舉辦選秀，要「創造」一個全新的偶像。
- **解釋**：使用 `new` 關鍵字呼叫建構函式。這會創出一個空物件，並把 `this` 綁在這個全新的物件上。
- **比喻**：新藝人剛出道，聚光燈照在「這個剛誕生的新人」身上。

```js
function Person(name) {
  this.name = name;
}

const bruce = new Person('Bruce');
console.log(bruce.name); // Bruce
```

### 5. 箭頭函式 (Arrow Function) —— **家教模式**

箭頭函式沒有自己的 this，它會繼承定義所在外層作用域的 this，且這個綁定無法被改變。這特別適合用在回調函式或巢狀函式中，避免 this 指向遺失。

- **情境**：藝人還小，家長（外層環境）去哪，他就跟著去哪。
- **解釋**：箭頭函式沒有自己的 `this`，它會直接繼承**定義時**外層環境的 `this`。一旦決定了，誰來點名（`call` / `bind`）都沒用。
- **比喻**：你是跟著父母出國的小孩，父母在哪一國（外層 this），你就在哪一國，別人叫不動你。

```js
const obj = {
  name: 'Bruce',
  delayedGreeting() {
    setTimeout(() => {
      console.log(`Hello, ${this.name}`); // Hello, Bruce
    }, 1000);
  }
};
obj.delayedGreeting();
```

## `this` 判斷順序（優先級由高到低）

判斷 `this` 時請依照以下順序檢查，優先級越高越先成立：

1. **箭頭函式**：直接使用定義時外層的 `this`，不受其他規則影響。
2. **new 綁定**：使用 `new` 呼叫建構函式。
3. **顯式綁定**：呼叫 `call()`、`apply()` 或 `bind()`。
4. **隱含綁定**：作為物件方法被呼叫。
5. **預設綁定**：以上皆不符合時的保底規則。

## **call、apply 和 bind 的差異與使用場景？**

| 方法 | 執行方式 | 參數傳遞方式 | 常見應用場景 |
| --- | --- | --- | --- |
| call | 立即執行 | 逐一傳入 | 借用其他物件方法、呼叫父類建構函式 |
| apply | 立即執行 | 以陣列形式傳入 | 處理大量參數（如 `Math.max`）、動態參數 |
| bind | 返回新函式 | 可預設部分參數 | 事件監聽器、回調函式固定 `this` |

```js
function introduce(hobby1, hobby2) {
  console.log(`I am ${this.name}, I like ${hobby1} and ${hobby2}`);
}

const person = { name: 'Bruce' };

introduce.call(person, 'coding', 'reading');    // 立即執行
introduce.apply(person, ['coding', 'reading']); // 立即執行
const bound = introduce.bind(person, 'coding');
bound('reading'); // 返回新函式，之後再呼叫

// 輸出結果:
// I am Bruce, I like coding and reading
// I am Bruce, I like coding and reading
// I am Bruce, I like coding and reading
```

## **箭頭函式與一般函式的 this 差異**

- **箭頭函式**：沒有自己的 `this`，繼承外層作用域，且無法透過 `call`、`apply`、`bind` 改變。
- **一般函式**：`this` 由呼叫方式決定，可被上述三種方法改變。

```js
const obj = {
  name: 'object',
  regularMethod() {
    setTimeout(function() {
      console.log(this.name); // undefined（this 變成 window）
    }, 100);
  },
  arrowMethod() {
    setTimeout(() => {
      console.log(this.name); // 'object'（繼承外層 this）
    }, 100);
  }
};

obj.regularMethod();
obj.arrowMethod();

// 輸出結果:
// undefined
// object
```

## **判斷下列程式碼中 this 的指向：**

```js
// 範例 1：隱含綁定
const obj = { name: 'object', method() { console.log(this.name); } };
obj.method(); // 'object'

// 範例 2：方法抽離
const method = obj.method;
method(); // undefined（預設綁定）

// 範例 3：箭頭函式回調
const arrowObj = {
  name: 'arrow',
  method() {
    setTimeout(() => {
      console.log(this.name);
    }, 100);
  }
};
arrowObj.method(); // 'arrow'

// 範例 4：new 綁定
function Person(name) {
  this.name = name;
  this.sayHi = function() {
    console.log('Hi, ' + this.name);
  };
}
const person = new Person('Bruce');
person.sayHi(); // 'Hi, Bruce'
```

### 最佳實踐建議

- 回調函式（`setTimeout`、`addEventListener` 等）優先使用箭頭函式，避免 `this` 指向錯誤。
- 在 Class 中，若方法需要作為回調使用，可在 constructor 內執行 `this.method = this.method.bind(this)`，或直接將方法寫成箭頭函式屬性。
- 遇到 `this` 問題時，先確認「這個函式是被誰、以什麼方式呼叫的」，再對照五大規則與優先順序即可快速定位。
