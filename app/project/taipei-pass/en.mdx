## Overview

Taipei City Government runs a constellation of digital services — a citizen ID app, an internal evaluation platform, paperless processing, a Zero Trust access portal — each built by a different team, often years apart. As Senior Frontend Engineer at Tradevan, I've worked on several of these over time, including building the TaipeiPASS app's digital certificate feature from scratch. The common thread across all of them: every system follows one shared design specification from the city's in-house design team, recognizable by the same blue-green palette and logo, and every system has to clear the same WCAG 2.1 AA accessibility bar.

## Challenge

Different codebases, different teams, different timelines — but citizens interacting with any of these services expect the same look, the same behavior, the same level of access. The city's designers held pixel-perfect fidelity as a non-negotiable standard, which made keeping that consistency intact across systems that were never built together harder than it sounds.

## What I Did

- Built the TaipeiPASS app's digital certificate feature (我的證書) end-to-end on the frontend
- Worked closely with the city's design team on every project, translating Figma specs into pixel-perfect production code — a standard they cared about deeply
- Implemented the shared design specification's components and WCAG 2.1 AA accessibility patterns across each ~5-person team I worked with
- Became the person teammates turned to for the trickiest frontend screens and accessibility edge cases, regardless of which system I was on

## Outcome

From citizen ID to paperless services to Zero Trust access, these systems still feel like they come from the same place — same visual language, same level of polish, same accessibility standard — even though no single team built them all.
