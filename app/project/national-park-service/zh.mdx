# 內政部國家公園署 — 系統化 React 視覺架構實踐

作為此專案的 **Lead Design Engineer**，我主導將國家公園署旗下多個核心分署的數位入口網，整合為一套具備高度一致性與擴充性的 **React Component Library**。我的核心任務是將厚重的靜態設計規範（Style Guide），轉譯為能直接服務於各分署的「動態開發體系」，涵蓋墾丁、玉山、陽明山等各大國家公園。

## 專案背景

本專案的核心目標是為國家公園署建立一套**「單一事實來源 (Single Source of Truth)」** — 一個能橫跨所有分署入口網站擴展，同時保留各國家公園獨特視覺識別的共用組件系統。

主要交付範疇：

- 基於 Atomic Design 原則的專屬 React Component Library
- 用於集中管理品牌的 SASS/CSS Design Token 系統
- 將無障礙標準（A11y AA）直接封裝入可重用組件
- 支援跨團隊協作的技術規格文件與實作指南

## 核心挑戰

**視覺破碎與技術債：** 各分署入口網過去由不同廠商開發，導致響應式（RWD）表現與互動邏輯存在顯著差異，且多依賴舊式技術棧。

**「規範」與「實作」的斷層：** 靜態 PDF 規範難以約束不同的開發團隊，導致旗下各分署門戶網站的視覺一致性與品質極難控管。

## 戰略解決方案

我採取了**「由下而上 (Bottom-up)」**的架構思維，為全署建立了一套**「單一事實來源」**的開發體系：

- **原子化組件開發 (Atomic Design)：** 將 UI 規範拆解為基礎原子組件（按鈕、輸入框、卡片），開發出專屬組件庫。這確保了無論是哪個分署系統調用，其視覺比例與互動行為皆能達成 **100% 一致**。
- **Design Tokens 系統化定義：** 將顏色、字體、間距定義為 **SASS/CSS 變數系統**。未來若要更新品牌色，只需改動一處變數，所有平台皆能自動同步，極大化降低維護成本。
- **無障礙 A11y 的底層封裝：** 將 ARIA labels 與鍵盤導航邏輯直接封裝在組件中。這讓其他開發人員即便不熟悉規範，只要調用組件，就能自動符合政府 **AA 級合規標準**。

## 核心貢獻

- **多主題彈性架構：** 建立支援多主題切換的架構，在維持品牌一致性的同時，保留各國家公園的地景視覺特色。
- **技術規格與實作指南：** 撰寫了詳盡的技術文件，包含 React 組件 API、SASS 邏輯以及 A11y 實作指南，確保跨團隊協作時的低門檻。
- **像素級還原：** 達成從 Figma 到 Production Code 的零誤差轉換，展現對細節與性能的極致要求。

## 最終成果

透過這個專案，我實現了：

- **加速開發速率：** 透過「即插即用」的組件架構，有效消除了重複性的 UI 開發工作，使後續各分署入口網站的交付流程變得更加敏捷。
- **數位資產整合：** 成功將分散的資源整合為一套**具備長期維護價值的數位資產**，並定義了後續相關部會技術選型的標準範本。
- **可複製的品質標準：** 證明了在正確的架構設計下，小型團隊也能在大規模多入口生態系中，貫徹一致、無障礙且像素級精準的 UI 品質。
