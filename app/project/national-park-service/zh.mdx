# 內政部國家公園署 — 系統化 React 視覺架構實踐

![image](/nps.png)
> **Project Link:** [https://www.nps.gov.tw/ch](https://www.nps.gov.tw/ch) ↗

## 專案背景

國家公園入口網站採用一套依照政府官方 UI 規範打造的 React 元件庫——遵循 atomic design 概念,色彩與間距定義為 design tokens。我以這個既有入口網為參照,獨立重新打造了另外兩個政府系統的前端:臺灣登山申請一站式服務網,以及內政部公園署內部稅務相關業務系統。

## 核心挑戰

這三個系統沒有共用同一份程式碼,各自是獨立的 repo、獨立的開發歷史。真正的挑戰在於,要讓兩個完全不同的舊前端,看起來、用起來都像跟國家公園入口網出自同一套設計系統——但完全沒有碰過它的程式碼,也沒有任何正式文件可以參考。

## 實際做的事

- 仔細研究國家公園入口網既有的元件跟 UI 規範,在沒有正式文件的情況下反推出它的開發慣例
- 把臺灣登山申請一站式服務網的前端整個打掉重寫,複製同樣的元件模式與 design tokens
- 內政部公園署內部稅務相關業務系統也是同樣處理——獨立打造,對齊同一套視覺標準

## 成果

三個各自獨立程式碼庫的政府平台,現在呈現出一致的視覺與互動體驗。
