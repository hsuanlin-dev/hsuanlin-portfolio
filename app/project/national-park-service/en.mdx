# National Park Service — Scalable UI Framework

![image](/nps-en.png)
> **Project Link:** [https://www.nps.gov.tw/en](https://www.nps.gov.tw/en) ↗

## Overview

The National Park Service portal runs on a React component library built around an official government UI specification — atomic design principles, with colors and spacing defined as design tokens. Using that portal as my reference, I independently rebuilt the frontends of two other government platforms: a one-stop mountain climbing permit application portal, and an internal land/tax-related system.

## Challenge

None of these three systems share a single codebase — each lives in its own repo, with its own history. The real challenge was making two completely different legacy frontends look and behave as if they came from the same design system as the National Park portal, without ever touching its actual code, and without any formal documentation to work from.

## What I Did

- Studied the National Park Service portal's existing components and UI specification closely, reverse-engineering its conventions since no formal documentation existed
- Rebuilt the mountain climbing permit application portal's frontend from scratch, replicating the same component patterns and design tokens
- Did the same for the internal land/tax-related system, building it independently to match the same visual standard

## Outcome

Three separate government platforms, each with its own codebase, now look and behave consistently.
