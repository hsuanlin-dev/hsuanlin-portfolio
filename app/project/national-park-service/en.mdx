# National Park Service — Scalable UI Framework

As the **Lead Design Engineer**, I spearheaded the integration of digital portals for multiple core agencies under the National Park Service into a unified, scalable **React Component Library**. My mission was to translate heavy, static Style Guides into a "dynamic development ecosystem" serving various national parks (e.g., Kenting, Yushan, Yangmingshan).

## Project Overview

This project involved building a **single source of truth** for the entire National Park Service agency — a shared component system that could scale across all sub-portals while preserving the unique identity of each national park.

Core deliverables include:

- A proprietary React Component Library based on Atomic Design principles
- A SASS/CSS Design Token system for centralized brand management
- Accessibility (A11y AA) encapsulated directly into reusable components
- Technical documentation and implementation guides for cross-team collaboration

## The Challenge

**Visual Fragmentation & Tech Debt:** Sub-portals were previously developed by different vendors, leading to significant discrepancies in RWD performance and interaction logic, often relying on legacy tech stacks.

**The Implementation Gap:** Static PDF specifications failed to constrain diverse development teams, making it extremely difficult to control visual consistency and output quality across agency-wide portals.

## Strategic Solution

I adopted a **"Bottom-up" architectural approach** to establish a **"Single Source of Truth"** for the entire agency:

- **Atomic Design in React:** Deconstructed UI specs into foundational atoms (Buttons, Inputs, Cards) to build a proprietary library, ensuring 100% consistency in visual proportions and behaviors across any sub-system.
- **Systematized Design Tokens:** Defined colors, typography scales, and spacing as a **SASS/CSS variable system**. This ensures that future brand updates require only a single change to sync across all connected platforms, minimizing maintenance costs.
- **Encapsulated A11y:** Embedded ARIA labels and **Keyboard Navigation** directly into React components, enabling developers to meet government **AA standards** by default without needing deep accessibility expertise.

## Key Contributions

- **Scalable & Multi-theme Architecture:** Built a flexible framework supporting multi-theme switching, maintaining brand consistency while preserving the unique landscapes of each national park.
- **Design System Specs & Implementation Guide:** Authored detailed technical documentation covering React component APIs, SASS logic, and A11y guides to ensure low-friction cross-team collaboration.
- **Pixel-Perfect Fidelity:** Achieved zero-error conversion from Figma mockups to production code, demonstrating a rigorous commitment to detail and performance.

## Impact & Results

Through this project, I delivered:

- **Accelerated Development Velocity:** By providing "plug-and-play" components, we eliminated redundant UI development, significantly streamlining the delivery pipeline for subsequent sub-portals.
- **Digital Asset Consolidation:** Successfully integrated scattered resources into a valuable, long-term digital asset, setting the standard for future technical selections within related government departments.
- **Replicable Quality Standard:** Proved that with the right architecture, a small team can enforce consistent, accessible, and pixel-perfect UI across a large-scale multi-portal ecosystem.
